## 決定事項（確認済み）

- **レベル選択**：トップ画面で「通常（1択）」「上級（2択）」を選んでスタート
- **採点**：2つとも正解で初めて正解（完全一致のみ、部分点なし）
- **回答操作**：2つ選んでから「回答する」確定ボタンで判定
- **出題数**：上級も通常と同じ10問（Good/Bad 各5問）
- **問題データ作成**：上級問題は**ユーザー本人が作成**（私はデータ形式の枠組みだけ用意）
- **選択操作**：選んだものはタップで選択解除（toggle）。**2つ選択した状態では他の選択肢を `disabled` にする**

## 全体像

`level`（`'normal'` / `'advanced'`）という状態を1つ追加し、トップ → クイズ → 結果へ受け渡す。
1択ロジックを壊さず、`correct_answer`（単数）と `correct_answers`（複数）の両対応にするのが基本方針。

**内部では常に「正解を配列で扱う」**と分岐が減る：
- 既存問題は読み出し時に `correct_answers ?? [correct_answer]` へ正規化
- 回答も常に配列 `selected: number[]`
- 正誤判定は「ソートして完全一致」の共通関数 `isExactMatch(selected, correct)` を1つ用意

これで通常・上級の分岐が「選択数の上限」と「即判定 or 確定ボタン」だけになる。

## 変更ファイルと内容

### 1. `src/data/questions.js` — データと抽出ロジック

**(a) 上級用の問題データを新規追加**
- 1つのシナリオが **2つの原則** に同時に該当／違反する事例を作成
  （例：「エラー時に専門用語のコードだけ表示し、復帰手段もない」→ 原則2＋原則9）
- 新フィールド `correct_answers: [2, 9]`（配列）。`type`（good/bad）は流用
- 既存の単数 `correct_answer` 問題はそのまま温存（通常レベル用）

```js
{
  id: 101,
  level: 'advanced',
  type: 'bad',
  situation: '...',
  correct_answers: [2, 9],   // ← 配列
  explanation: '...',
}
```

**(b) 抽出関数をレベル対応に**
- `selectQuestions(level)` に引数追加
  - `normal`：現状どおり単数問題からGood/Bad各5問
  - `advanced`：`correct_answers` を持つ問題からGood/Bad各5問

> 留意点：上級問題のシナリオ本文はユーザーが作成する。私はデータ形式（`correct_answers` 配列を持つ雛形）と、空でも動く受け皿を用意する。Good/Bad 各5問が揃うまでは仮データで動作確認する。

### 2. `src/App.jsx` — レベル状態の管理
- `const [level, setLevel] = useState('normal')` を追加
- `handleStart(selectedLevel)` で `level` を保存し `selectQuestions(selectedLevel)` を呼ぶ
- `QuizScreen` と `ResultScreen` に `level` を props で渡す

### 3. `src/components/TopScreen.jsx` — レベル選択UI
- 「クイズを始める」を、レベル選択（通常／上級）→ スタート に変更
  - 案：2ボタンを並べ、それぞれ `onStart('normal')` / `onStart('advanced')` を呼ぶ
- 上級の説明文（「2つの原則に該当します。両方選んでください」）を添える
- 既存の「10原則とは？」モーダルはそのまま

### 4. `src/components/QuizScreen.jsx` — 複数選択ロジック（中心的変更）
- `selectedAnswer`（単数）→ `selected`（配列）に変更
- **通常レベル**：従来どおりタップで即判定（`level === 'normal'`）
- **上級レベル**：
  - タップで選択トグル（再タップで解除）
  - **2つ選択した状態になったら、未選択の選択肢を `disabled` にする**（3つ目を選べない）
  - 2つ選択中のみ「回答する」ボタンを活性化 → 押下で判定
  - 判定：選択2つの集合が `correct_answers` と完全一致なら正解
  - 見た目：選択中（青）、2択到達時は未選択を `disabled`（淡色）、判定後は正解の2つに✓、誤選択に✗
- `answersRef` のデータを `{ question, selected: [...], isCorrect }` に拡張（通常も配列化して統一）
- `getButtonState` を配列・複数正解に対応するよう書き換え
- 問題文：上級は「該当する原則を**2つ**選んでください」を明示

### 5. `src/components/ResultScreen.jsx` — 複数回答の表示
- `a.selected` / 正解側が配列になり得る前提で表示調整
  - 正解原則：`correct_answers ?? [correct_answer]` で正規化して複数表示
  - あなたの回答：配列を「原則X・原則Y」のように連結表示
- スコア集計（`isCorrect` のbool集計）は完全一致判定なのでロジック変更不要
- 称号（RANKS）：上級用に別メッセージを用意するかは任意（まずは流用可）

### 6. CSS（`QuizScreen.module.css` 等）
- 「選択中」状態のスタイル追加（判定前の青ハイライト）
- 「回答する」確定ボタンのスタイル（非活性／活性）

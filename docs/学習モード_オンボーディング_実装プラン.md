# 学習モード（オンボーディング）追加プラン

## Context（背景）

現在 `ux-quiz` は TOP →（通常 / 上級モード）→ クイズ → 結果 という流れで、
「ニールセンの10原則とは？」を押すと **TOP上でモーダル** が開いて10原則の一覧を確認できる。

やりたいこと:
1. TOP の「ニールセンの10原則とは？」を押したら、**オンボーディング形式の学習モード**（スライド）を開始する。
   - 1枚目: ニールセンの10原則の**概要**説明
   - 2枚目以降: 1原則 = 1スライドで「原則名・概要・例示」を提示
2. 今 TOP に出している**モーダルはクイズ中（通常/上級）から確認できる導線**へ移し、
   クイズ画面のボタンを押したらモーダルで10原則を確認できるようにする。

確認済みの方針:
- **例示の内容はユーザーが用意する**（データ構造だけ用意し、本文は後でユーザーが記入）
- **学習モード完了後は TOP に戻る**（クイズへの自動導線は付けない）

## 現状把握（関連ファイル）

- `src/App.jsx` — `screen` state（`'top' | 'quiz' | 'result'`）で画面切替。`handleStart(level)` でクイズ開始。
- `src/components/TopScreen.jsx` — `showModal` state でモーダル開閉。`HEURISTICS.map` でモーダル本体を描画（L42-76）。「ニールセンの10原則とは？」ボタン（L37-39）。
- `src/components/TopScreen.module.css` — モーダル関連CSS（`.overlay` `.modal` `.modalHeader` `.modalBody` `.modalSummary` `.modalLink` `.hItem` 系、L169-317）。
- `src/data/questions.js` — `HEURISTICS` 配列（`id, name, en, description`）。`example` フィールドは無い。
- `src/components/QuizScreen.jsx` — ヘッダーに「やめる」ボタン（L79-81）。ここに導線を追加する。
- `src/index.css` — グローバルCSS変数（`--color-primary` 等）と `fade-in` / `slide-up` アニメーション（既存利用可）。

## 実装方針

### 1. データ拡張 — `src/data/questions.js`
`HEURISTICS` の各要素に `example` フィールドを追加（**本文はユーザー記入用のプレースホルダー**）。
```js
{ id: 1, name: '...', en: '...', description: '...', example: '' },  // example はユーザーが後で記入
```
- 空文字の間も LearnScreen 側で例示ブロックを条件表示し、崩れないようにする。

### 2. モーダルを共有コンポーネント化 — 新規 `src/components/HeuristicsModal.jsx` + `.module.css`
- 現在 `TopScreen.jsx` 内にインラインで書かれているモーダル（概要文 + nngroupリンク + 10原則リスト）を**そのまま切り出して再利用可能なコンポーネント**にする。
- Props: `onClose`。`createPortal` で `document.body` に描画（既存実装踏襲）。
- 対応する CSS（`.overlay` `.modal` `.modalHeader` `.closeBtn` `.modalBody` `.modalSummary` `.modalLink` `.hItem*`）を `TopScreen.module.css` から `HeuristicsModal.module.css` へ移設。

### 3. 学習モード（スライド）— 新規 `src/components/LearnScreen.jsx` + `.module.css`
- Props: `onExit`（TOPに戻る）。
- スライド構成: 全11枚 = 概要1枚 + 原則10枚。`useState` で `step`（0〜10）管理。
  - `step === 0`: 概要スライド（HeuristicsModal の概要文と同じ趣旨の説明 + nngroupリンク）。
  - `step 1〜10`: `HEURISTICS[step-1]` を表示（番号バッジ・原則名・英名・`description`・`example`）。
- ナビゲーション:
  - 「戻る」（step>0 で有効） / 「次へ」（step<10）。
  - 進捗インジケータ（例: ドット or 「n / 11」）。
  - 右上に「✕ / TOPに戻る」終了導線。
  - 最終スライド（step===10）では「次へ」を **「TOPに戻る」** に切替（→ `onExit`）。
- スタイルは TopScreen/QuizScreen のトーン（カード・余白・`fade-in`/`slide-up`）に合わせる。スライド切替時にアニメーション付与。

### 4. ルーティング — `src/App.jsx`
- `screen` に `'learn'` を追加。
- `handleLearn = () => setScreen('learn')`、`handleExitLearn = () => setScreen('top')` を追加。
- `{screen === 'learn' && <LearnScreen onExit={handleExitLearn} />}` を追加。
- `<TopScreen onStart={handleStart} onLearn={handleLearn} />` に変更。

### 5. TOP からモーダル削除・学習モード起動 — `src/components/TopScreen.jsx`
- `onLearn` を props に追加。
- 「ニールセンの10原則とは？」ボタンの `onClick` を `setShowModal(true)` → `onLearn()` に変更。
- `showModal` state とインラインのモーダル JSX（L42-76）、`createPortal`/`useState` import を削除（モーダルは HeuristicsModal へ移管済み）。
- `.hintBtn` のスタイルはそのまま流用。

### 6. クイズ中の確認導線 — `src/components/QuizScreen.jsx` + `.module.css`
- `showRef` state を追加。
- ヘッダーの「やめる」横（L79付近）に確認ボタンを追加（例: 「10原則を見る」/ 📖 アイコン）。
- クリックで `<HeuristicsModal onClose={() => setShowRef(false)} />` を表示。
- ボタン用の軽量スタイルを `QuizScreen.module.css` に追加（既存 `.quitBtn` を踏襲）。

## 影響範囲まとめ
- 新規: `HeuristicsModal.jsx/.module.css`, `LearnScreen.jsx/.module.css`
- 変更: `App.jsx`, `TopScreen.jsx`, `TopScreen.module.css`, `QuizScreen.jsx`, `QuizScreen.module.css`, `data/questions.js`
- クイズ・結果ロジック（`selectQuestions`, 採点）には**変更なし**。

## 検証方法
1. `npm run dev` で起動。
2. TOP の「ニールセンの10原則とは？」→ 学習モード（概要→原則1→…→原則10）がスライドで進む / 戻れる。
3. 最終スライドの「TOPに戻る」で TOP に戻る。✕ でも途中離脱できる。
4. 通常モード・上級モードでクイズ開始 → ヘッダーの確認ボタンでモーダルが開き、10原則を確認 → 閉じても回答状態が保持される。
5. `npm run build` がエラーなく通ることを確認。

## 補足
- `example` の本文は空のまま実装し、ユーザーが `HEURISTICS` の各 `example` に記入する想定（記入後そのままスライドに反映）。

## 1. 前提・制約

- **色**: 現状のインディゴ基調を据え置き（色相は変えない）
- **フォント**: systemフォントを使う（軽量重視 / Webフォントは読み込まない）
- **アニメーション**: 無駄な動きは避ける。装飾的なモーションは足さない方針
- **方針の軸**: フォントや動きに頼らず、**レイアウト・階層・余白・モチーフ**で印象づける
- 既存の作り: React + Vite + CSS Modules。デザイントークンは `src/index.css` の `:root` に集約済み

### 今フォーカスしたいこと
細部（フォーカスリング・余白微調整など）ではなく、**サービスを印象づける大きめの部分**:
- サービスの"顔"になるモチーフの追加
- レイアウトそのものの変更

## 2. モチーフの種 ＝「10本ノック」という名前

サービス名「**ヒューリスティクス10本ノック**」自体が、デザインの起点になる。

- 「10本ノック」＝野球などの**反復練習ドリル**を指す言葉
- → このサービスの性格は「**UXを反復で叩き込む特訓**」。名前からこの個性を持っている
- ここを視覚化すると"らしさ"が出る

### 引き出せるモチーフ候補

| モチーフ | 内容 | 再利用性 |
|---|---|---|
| **「10」という数字** | 10原則・全10問・10本ノック、すべて 10 に収束する。数字をヒーローにできる | TOPのヒーロー |
| **10個のユニット（的/マス/ドット）** | ニールセンの10原則を「10個の視覚要素」で表現 | TOP・Quiz進捗・Resultで再利用でき、**全画面を貫くアイデンティティ**になりうる |
| **ドリル/特訓のトーン** | ナンバリング、スコアボード感、反復の気配 | 全体の世界観 |

## 3. レイアウト方向性（3案）

systemフォント・静的でも成立する範囲で、印象が大きく変わる3パターン。

### 案A: 巨大な「10」をヒーローに
名前の「10」を画面の主役にしたタイポグラフィ中心のデザイン。インパクト重視で軽量。

```
┌──────────────────┐
│ NIELSEN'S 10 HEURISTICS│
│                  │
│    ██   ███      │ ←巨大な 10
│    ██  ██ ██     │
│    ██   ███      │
│  本ノック          │
│  ヒューリスティクス  │
│                  │
│  UIシナリオを見て…  │
│                  │
│  [ 通常モード    › ]│
│  [ 上級モード    › ]│
└──────────────────┘
```

- ◎ 一番インパクトが出る / 軽量 / モバイルでも成立
- △ 「10」をどう美しく組むか（systemフォントでの数字の見え方）が肝
- 再利用: Result の点数表示と数字トーンを揃えられる

### 案B: 10原則グリッドを象徴に
10原則を「10個の番号マス」で表現し、全画面で再利用できる象徴にする。

```
┌──────────────────┐
│ ヒューリスティクス10本ノック│
│  UIシナリオを見て…  │
│                  │
│  ① ② ③ ④ ⑤      │ ←10原則を
│  ⑥ ⑦ ⑧ ⑨ ⑩      │  10マスで表現
│                  │
│  [ 通常モード      ]│
│  [ 上級モード      ]│
│  ニールセンの10原則とは?│
└──────────────────┘
```

- ◎ 学習感・網羅感が出る / モチーフを全画面（Quiz進捗・Result）に展開できる
- ◎ 「何を学ぶアプリか」が一目で伝わる
- △ 情報量が増える分、整理しないと散らかる

### 案C: 2カラム / スプリットレイアウト
左にコピー+CTA、右に10原則のビジュアル。横幅を使うモダンなLP風。

```
┌─────────┬────────┐
│ 10本ノック │ 1 視認性 │
│         │ 2 一致   │
│ UIを見て │ 3 自由度 │
│ 当てよう │ 4 一貫性 │
│         │ …(10原則)│
│ [通常]   │         │
│ [上級]   │         │
└─────────┴────────┘
```

- ◎ PCで映える / 情報を一望できる
- △ このアプリはモバイル中心（既存コンテナ max-width 480〜600）。PC専用の作り込みが増える
- △ モバイルでは結局1カラムに畳む必要 → 実質2レイアウト分の管理

### デザイン言語（確定の方向）

> **「エディトリアル × ざらつき（リソグラフ風）× モダン幾何学」 ─ 賢さ6 : 遊び4**

| 軸 | 中身（回答の根拠） |

### モチーフの収束（推奨）

**「10個の幾何学シェイプ ＝ 10原則」システム** を推奨。

- 各原則に固有の幾何学シェイプを1つ割り当てる（計10種）
- **TOP**: 10シェイプを"顔"として配置（散りばめ or グリッド）→ サービスの象徴に
- **Quiz / Result**: その設問の原則シェイプをワンポイントで使用 → 回答画面の没個性を回避（Q5を解決）
- grain質感をシェイプに乗せて chill・エディトリアルに
- → golfイラスト案より「概念的な整合（10＝10原則）」と「全画面での再利用性」で優位。温かみは grain＋暖色アクセントで補う

### パレットの調整方針

- **インディゴ**: 賢さの軸として維持
- **背景**: 寒色グレー `#f4f5f7` を**維持**（Q12）。温かみは色でなく grain・モチーフ・控えめな暖色で出す
- **暖色アクセント1色**: 足してよいが**控えめに**（Q13）。色味はおまかせ寄り → grain と相性のよい少しくすんだ暖色を想定（テラコッタ/マスタード系を候補に試す）
- **質感**: 幾何学シェイプは**フラット塗り＋grain**（Q14）

### Q11の決め方（メモ）
他の回答（背景は寒色維持・アクセントは控えめ・フラット＋grain）はいずれも「**抑制された系統的なルック**」を指している。ここに全面ベクターイラスト（例: golf）を置くと、唯一の強い暖色要素になり「控えめ」と衝突＆各画面での再利用も難しい。
→ 整合するのは **① 幾何学シェイプ系**。TOPでは10シェイプを**構成的に配置して"ゆるい情景"のように見せれば**、イラスト的なリッチさも兼ねられる（イラストの恣意性・再利用問題を回避）。
→ 迷いは「見れば解ける」ので、**①でTOPの静的プロトタイプを作って判断**する。イラスト欲が残れば②を試す。

- 補助: ライトグレーのシェイプ（半円など）で抜け感

**モチーフ＝幾何学シェイプ（フラット＋grain）**

- バッジ: アウトラインのピル（インディゴ枠）「NIELSEN'S 10 HEURISTICS」

- ボタン:

- ヒントリンク: 下線つきインディゴ「ニールセンの10原則とは？」

## 10. 実装アプローチ（軽量・静的前提）

- **グレイン**: SVG `feTurbulence` のノイズを生成し、固定オーバーレイ（低不透明度）で全面に敷く。または小さなノイズPNGをタイル。**静的・軽量**（アニメなし）
- **シェイプ**: 装飾用の SVG/CSS レイヤーを1枚、`position:absolute` で**コンテンツの背面**（`z-index:0`, `pointer-events:none`, `aria-hidden`）に配置。シェイプにもグレインを適用
- **カラートークン**: `--color-accent-terracotta` / `--color-accent-mustard` を `:root` に追加
- **レスポンシブ／可読性**: 装飾は上下に寄せ、中央帯は空ける。画面が小さい時はシェイプを間引く。テキストの背面に重なる場合はコントラストを担保
- **ボタン**: JSX を「アイコン＋テキスト塊（＋バッジ）」の横並び構成に作り替え
- **アニメ整理**: 既存 stagger slide-up は撤去し、控えめな fade-in のみ（Q7）
